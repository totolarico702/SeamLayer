The Problem

HTML was built for eyes.
Not for agents.

The web as it exists today forces AI agents to do absurd amounts of work just to read content — and gives publishers nothing in return.

Today — HTML Scraping
Publishers lose.
Agents waste compute.
  • ✕ 59% of HTML tokens are structural noise: scripts, nav, ads, cookie banners
  • ✕ 4 separate LLM calls needed to classify, extract entities, infer metadata, and resolve action intents
  • ✕ No provenance — agents cannot verify content authenticity or attribution
  • ✕ Publishers receive zero compensation for AI consumption of their work
  • ✕ 72% hallucination risk from ambiguous, unclassified source data
With SeamLayer — .spk Protocol
Publishers control.
Agents get clean data.
  • ✓ Content pre-classified at source: Fact, Opinion, Statistic, Code — zero inference needed
  • ✓ 0 LLM calls to parse content: schema-directed reading replaces all inference steps
  • ✓ Ed25519 cryptographic signature on every document — tamper-proof provenance
  • ✓ Token-based micropayments per depth level — publishers monetize AI consumption
  • ✓ 18% hallucination risk — publisher-classified, signed, versioned content
The Protocol

Three files.
One protocol.

SeamLayer defines a parallel layer to the human web — three file formats that together give AI agents everything they need to discover, verify, consume, and pay for structured knowledge.

.spsk
Spark Protocol Site Key
The public entry point. Served at /.well-known/spark.spsk. Contains publisher identity, Ed25519 public keys, and pointers to content resources.
publisheridentity + contact
public_keysEd25519 verification
entry_pointscontent index URLs
compliance_requiredlevel_0 → level_3
.spk
Spark Protocol Knowledge
The signed knowledge document. Depth-partitioned content graph with provenance metadata, usage rights, and cryptographic integrity proof. One file per content resource.
depth_layers0→N with access rules
spark:sectionTypeFact / Opinion / Code…
usagetrain / finetune / rag
integrity.signatureEd25519 signed
.spki
Spark Protocol Knowledge Interface
The action surface. Declares what autonomous agents can do — search, purchase, subscribe, report — with explicit intents, costs, and compliance requirements. Zero DOM inference.

from seamlayer_skill import SeamLayerSkill, TokenWallet # Initialize with token budget wallet = TokenWallet(balance=500, agent_id="spark:agent:my-agent") spark = SeamLayerSkill(wallet=wallet) # 1. Discover — is this domain SeamLayer-enabled? site_key = spark.discover("example-media.com") # 2. Search the knowledge graph results = spark.search("example-media.com", "agent protocols 2026") # 3. Consume content (signature auto-verified) content = spark.consume(results[0].content_id, depth=2) # 4. Get LLM-ready context block print(content.to_context_block()) # Wallet auto-debited, receipt stored print(wallet.statement())
from seamlayer_skill import SeamLayerSkill, TokenWallet, as_langgraph_tools from langgraph.prebuilt import create_react_agent # Create SeamLayer tools wallet = TokenWallet(balance=1000) skill = SeamLayerSkill(wallet=wallet) tools = as_langgraph_tools(skill) # tools is a list of 4 callables: # spark_discover(domain) # spark_search(domain, query) # spark_consume(content_id, depth) # spark_wallet_balance() # Plug into any LangGraph agent agent = create_react_agent( model=llm, tools=[*other_tools, *tools] ) # Agent now automatically discovers SeamLayer domains # and prefers structured content over HTML scraping result = agent.invoke({"messages": [("user", "Research AI protocols on example-media.com")]})
